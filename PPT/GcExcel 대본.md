# GcExcel 대본

(1페이지)

- 안녕하세요. GrapeCity MVP 이상준입니다.
이번 시간에는 GcExcel로 리눅스에서 Excel을 처리하는 방법에 대해 소개하겠습니다.

(2페이지)

- 엑셀은 오늘날 아직도 기업에서 가장 널리 사용되는 도구일 것입니다.
개발자가 없는 부서와 서로 문서를 주고 받을 때 엑셀이 이용되고는 하는데요.
아마 표 형태의 정형화된 데이터를 표현하기에 가장 적합하기 때문이 아닐까 생각합니다.
그러다보니 엑셀을 통해서 데이터를 대량으로 업로드 한다거나, 대량으로 엑셀파일을 만들어낸다거나 하는 작업을
심심치 않게 할 수 있습니다.

(3페이지)

엑셀을 잘 다룰 수 있는 개발자도 분명 계시겠지만, 저는 엑셀을 잘 못하기 때문에 엑셀을 잘하는 부서에서
엑셀문서를 보내줬을 때 그 디자인과 수식들을 모두 살려서 엑셀을 조작하지는 못합니다.
주로 해왔던 업무도 엑셀데이터를 전달받고, 시트 안에있는 데이터를 모두 반복문으로 불러와서
객체화하고 DB에 저장하거나 처리하거나...주로 그런 방식으로만 했었는데요.

아마도 엑셀을 주로 사용하는 부서가 사내에 많을 수록 엑셀에 대한 어려운 처리가 불가피해질 것입니다.
차라리 C# 코드로 필요한 로직을 짜내서 처리하는 것은 가능하겠지만, 그냥 프로그램에서 계산된 데이터를 엑셀 표에 채워넣는 것이 전부일 것이고, 셀마다 수식을 넣거나 디자인을 한다거나 하는 것은 좀 어려울 것 같습니다.
물론 Office 라이브러리를 사용하면 되겠지만, 널리 알려져 있듯 Office 라이브러리는 버전을 타는데다, 엄청나게 느립니다.

그러면 정말 답이 없을까요?

(4페이지 1초, 5페이지)

GcExcel은 GrapeCity에서 개발한 고성능 엑셀 처리 라이브러리 입니다.
최신 .NET 버전에 맞춰 계속 업데이트 되기 때문에 현재 .NET이 크로스플랫폼이 가능한만큼, 당연히 윈도우말고도 다른 플랫폼에서 GcExcel을 통한 고성능 Excel처리가 가능합니다.

오늘 이것을 Azure에서 Linux 기반의 App Service에 올려서 타사 엑셀 처리 라이브러리와 처리속도를 비교해 볼 것입니다.

(6페이지)

현재 제가 Azure에 구성한 하드웨어는 Azure App Service Linux Plan B2 입니다.
Azure App Service Linux Plan B2는 CPU Core 2개와 Memory 3.5 GB, 저장소 10GB로 구성되어 있습니다.
